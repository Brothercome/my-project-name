Backend Engineer

도르팀의 Backend Engineer는 게이머가 자신의 플레이를 더 많이 보고, 반응하고, 연결되고, 다시 돌아오게 만드는 역할을 합니다. 피드, 프로필, 팔로우, 댓글, 좋아요, 알림, 공유 등 핵심 기능의 API와 비즈니스 로직을 구현합니다.

지원하기 커피챗 신청

주요업무

도르팀의 Backend Engineer 는 게이머가 자신의 플레이를 더 많이 보고, 반응하고, 연결되고, 다시 돌아오게 만드는 역할을 합니다.

Community Squad는 도르의 핵심 가치인 "좋은 순간이 기록되는 것을 넘어, 사람들 사이에 공유되고 관계로 이어지게 만든다"는 문제를 가장 앞에서 푸는 조직입니다.

Backend Engineer는 이런 일을 합니다

  • 커뮤니티 기능 전반의 서버와 데이터 구조를 설계하고 개발합니다.
  • 피드, 프로필, 팔로우, 댓글, 좋아요, 알림, 공유 등 핵심 기능의 API와 비즈니스 로직을 구현합니다.
  • 유저 간 관계가 자연스럽게 형성되고 유지될 수 있도록 데이터 모델과 시스템을 설계합니다.
  • 참여도와 리텐션을 높이기 위한 기능을 빠르게 실험하고 개선합니다.
  • 대규모 콘텐츠 소비와 반응이 안정적으로 처리될 수 있도록 성능과 안정성을 개선합니다.
  • PO, Product Designer, Frontend Engineer, Data Analyst와 함께 문제를 정의하고 실제 유저 행동의 변화를 만들어냅니다.
  • 단순한 기능 개발이 아니라, 커뮤니티와 네트워크를 제품 안에서 작동하게 만드는 역할을 맡습니다.

팀에서 만들어갈 서비스

피드 (Feed)

  • 유저가 더 자주 들어오고 더 오래 머물 수 있는 피드 경험을 만듭니다.
  • 어떤 영상이 소비되고, 어떤 영상이 반응을 만들고, 어떤 흐름이 반복 방문으로 이어지는지 개선합니다.

프로필 (Profile)

  • 유저가 자신을 표현하고, 다른 유저를 발견하고, 관계를 이어갈 수 있는 프로필 경험을 만듭니다.
  • 생산자와 소비자가 더 자연스럽게 연결되는 구조를 설계합니다.

소셜 액션 (Social Interaction)

  • 좋아요, 댓글, 팔로우, 공유, 저장 등 유저 간 상호작용을 설계합니다.
  • 단순 소비를 넘어 실제 반응과 관계가 생기도록 만드는 것이 목표입니다.

알림 (Notification)

  • 유저가 다시 돌아올 이유를 만드는 알림 경험을 설계합니다.
  • 팔로우한 유저의 활동, 내 콘텐츠에 대한 반응, 커뮤니티 내 중요한 변화가 자연스럽게 재방문으로 이어지도록 만듭니다.

네트워크와 커뮤니티 경험

  • 유저들이 서로 연결되고, 다시 만나고, 더 자주 머무르게 하는 구조를 만듭니다.
  • 단순한 영상 툴을 넘어 커뮤니티 제품으로 진화시키는 경험을 설계합니다.

Community Squad가 중요하게 보는 지표

  • Community 유저 수
  • Daily Retention / D7 / D30 Retention
  • 피드 시청 시간
  • 좋아요 / 댓글 / 공유 / 저장 등 참여 지표
  • 팔로우 수 / 관계 형성 지표
  • 알림 클릭률 및 재방문율
  • 생산자와 소비자 간 연결 밀도
  • 네트워크 효과 관련 지표

단순히 기능을 많이 만드는 것이 아니라, 더 많은 유저가 더 자주 들어오고, 더 많이 반응하고, 더 강하게 연결되도록 만드는 것이 핵심 목표입니다.

우리 팀이 일하는 방식

  • 도르팀은 대표가 원하는 제품이 아니라 고객이 원하는 제품을 만듭니다.
  • 1~2주의 짧은 이터레이션 안에서 문제 정의, 가설 수립, 실행, 측정, 학습을 반복합니다.
  • 데이터와 유저의 목소리를 바탕으로 자율과 책임을 가지고 제품을 만듭니다.
  • 엔지니어는 주어진 명세만 구현하는 역할이 아니라, 문제를 정의하고 결과를 만드는 역할입니다.
  • 빠르게 만들고, 빠르게 검증하고, 빠르게 방향을 조정합니다.
  • 기술적 완성도와 실제 사용자 가치를 동시에 높이는 데 집요하게 몰입합니다.

기술 스택과 툴

모든 스택에 대한 경험을 기대하지 않습니다. 다만 빨리 배우고 습득이 가능하기를 기대합니다.

  • Python
  • Node.js
  • TypeScript
  • MySQL
  • API / Backend Architecture
  • Analytics / Experimentation Tools

자격 요건

  • 실사용자가 있는 서비스의 백엔드 개발 경험이 있는 분
  • 하나 이상의 서비스를 출시하고, 사용자의 VOC를 기반으로 이터레이션해본 경험이 있는 분
  • Python, Node.js, TypeScript, MySQL 등으로 실제 제품을 개발할 수 있는 분
  • 데이터 모델링과 API 설계에 대한 이해가 있는 분
  • 기능 구현을 넘어 제품과 비즈니스 관점에서 문제를 해결할 수 있는 분
  • 고객의 목소리와 데이터를 기반으로 제품을 개선해본 경험이 있는 분
  • 어려운 문제를 끝까지 파고들어 해결하려는 집요함이 있는 분

우대 사항

  • San Francisco, CA / 미국 근무가 가능한 분
  • 소셜미디어, 커뮤니티, UGC 서비스의 백엔드 개발 경험이 있는 분
  • 피드, 프로필, 알림, 댓글, 좋아요, 팔로우, 추천 등 커뮤니티 기능을 개발해본 경험이 있는 분
  • 참여도, 리텐션, 재방문과 관련된 문제를 시스템으로 풀어본 경험이 있는 분
  • 대규모 트래픽이나 많은 유저 액션을 안정적으로 처리해본 경험이 있는 분
  • 캐시, 큐, 비동기 처리, 알림 시스템 설계 경험이 있는 분
  • 추천/랭킹/피드 노출 구조에 대한 이해가 있는 분
  • 애자일한 환경에서 빠르게 실험하고 개선해본 경험이 있는 분
  • 영어로 자유롭게 의사소통이 가능한 분
  • 게이머이거나 게임 유저 행동에 대한 높은 이해가 있는 분
  • 사람들이 왜 콘텐츠를 소비하고, 왜 반응하고, 왜 다시 돌아오는지에 관심이 많은 분

이런 분과 잘 맞아요

  • 단순히 API를 만드는 사람보다 제품 문제를 푸는 사람
  • 유저 간 관계가 형성되는 구조를 만드는 데 흥미가 있는 사람
  • 빠르게 만들고 빠르게 검증하는 환경을 선호하는 사람
  • 데이터 구조와 시스템 설계로 리텐션과 참여를 만드는 데 흥미가 있는 사람
  • 기술적 완성도뿐 아니라 실제 유저 행동 변화를 중요하게 보는 사람
  • 단순한 기능이 아니라 사람들이 계속 돌아오고 싶어지는 제품을 만들고 싶은 사람

  • 백엔드 구조나 데이터 모델을 통해 제품 경험을 개선한 사례가 있다면 구체적으로 적어주세요.
  • 데이터와 VOC를 기반으로 스스로 정의하고 해결한 문제가 있다면 알려주세요.

  • 성능, 안정성, 확장성 문제를 해결한 경험이 있다면 수치와 함께 설명해주세요.
