Data Analyst

도르팀의 Data Analyst는 게이머가 자신의 플레이를 더 쉽게 기록하고, 더 자주 만들고, 더 많이 공유하고, 더 자주 돌아오도록 제품의 핵심 의사결정을 데이터로 이끄는 역할을 합니다. Creator, Community, Growth Squad와 긴밀하게 협업합니다.

지원하기 커피챗 신청
주요 업무

이 포지션은 특정 하나의 기능만 분석하는 역할이 아닙니다. 도르의 핵심 제품 경험 전반을 보며, 유저가 왜 만들고, 왜 공유하고, 왜 반응하고, 왜 다시 돌아오는지를 구조적으로 파악하고 제품팀이 더 좋은 판단을 할 수 있도록 돕습니다.

  • 자동 녹화, 클립 생성, 편집, 공유, 피드, 프로필, 반응, 알림 등 도르의 핵심 유저 행동 데이터를 분석합니다.
  • 유저가 어디서 이탈하는지, 왜 반복 사용하지 않는지, 무엇이 공유와 재방문으로 이어지는지 구조적으로 파악합니다.
  • Creator / Community 전반의 핵심 KPI를 정의하고, 지속적으로 측정 가능한 형태로 관리합니다.
  • 제품팀과 함께 실험을 설계하고, 어떤 변화가 실제로 유의미한 개선을 만들었는지 분석합니다.
  • VOC와 정량 데이터를 함께 보며 제품 우선순위 수립에 필요한 인사이트를 도출합니다.
  • 대시보드, 분석 문서, 리포트를 통해 팀이 더 빠르게 판단하고 실행할 수 있도록 돕습니다.
  • 이벤트 설계, 데이터 품질 검증, 분석 환경 개선에도 함께 관여합니다.
이 포지션에서 특히 중요한 문제

이 역할은 숫자를 정리하는 사람보다 가장 중요한 제품 질문을 데이터로 밝혀내는 사람을 찾고 있습니다.

  • 유저는 왜 첫 영상을 만들고, 왜 그 다음에는 만들지 않을까?
  • 어떤 경험이 생성 완료, 공유, 시청으로 이어질까?
  • 어떤 유저가 더 오래 남고, 어떤 유저가 더 빨리 떠날까?
  • 피드와 반응 경험은 실제 재방문과 리텐션에 어떤 영향을 줄까?
  • 어떤 실험이 실제로 유의미한 행동 변화를 만들었을까?
  • 제품팀이 지금 가장 먼저 풀어야 할 문제는 무엇일까?
팀에서 만들어갈 서비스
게임 플레이 자동 녹화 (Record)
  • 유저가 놓치고 싶지 않은 순간을 자동으로 기록할 수 있도록 만듭니다.
  • 자동 녹화 경험이 실제 사용과 반복 생성으로 이어지는지 분석합니다.
게임 영상 생성 / 편집 / 공유 (Create / Edit / Share)
  • 유저가 더 쉽게 영상을 만들고, 더 자주 공유하도록 만드는 경험을 분석합니다.
  • 생성 → 편집 → 공유까지 이어지는 전체 퍼널을 구조적으로 파악합니다.
커뮤니티 (Community)
  • 유저가 더 자주 들어오고, 더 많이 보고, 더 많이 반응하고, 더 오래 남는 경험을 분석합니다.
  • 피드, 프로필, 댓글, 좋아요, 팔로우, 알림이 실제 리텐션과 참여에 어떤 영향을 주는지 분석합니다.
도르팀이 중요하게 보는 지표
  • New Daily Acquisition User
  • Daily Retention / D7 / D30 Retention
  • Community engagement

더 많은 유저가 더 자주 들어오고, 더 많이 반응하고, 더 강하게 연결되도록 만드는 것이 핵심 목표입니다.

우리 팀이 일하는 방식
  • 도르팀은 대표가 원하는 제품이 아니라 고객이 원하는 제품을 만듭니다.
  • 1~2주의 짧은 이터레이션 안에서 문제 정의, 가설 수립, 실행, 측정, 학습을 반복합니다.
  • 데이터 분석가는 결과를 정리하는 역할이 아니라, 문제를 정의하고 의사결정을 더 날카롭게 만드는 역할입니다.
  • 감이 아니라 데이터와 고객의 목소리를 바탕으로 판단합니다.
  • 빠르게 분석하고, 빠르게 공유하고, 빠르게 다음 액션으로 연결합니다.
  • 유저가 사랑할 수 있는 제품을 만드는 데 필요한 가장 중요한 질문을 함께 던집니다.
기술 스택

모든 스택에 대한 경험을 기대하지 않습니다. 다만 빨리 배우고 습득이 가능하기를 기대합니다.

  • SQL
  • BigQuery 또는 데이터 웨어하우스
  • Product Analytics Tool
  • Dashboarding / BI Tool
  • A/B Test Analysis
  • Mixpanel
자격 요건
  • 실사용자가 있는 서비스의 데이터를 분석해본 경험이 있는 분
  • SQL을 활용해 데이터를 직접 추출하고 분석할 수 있는 분
  • 하나 이상의 서비스를 분석하고, 이를 바탕으로 제품 개선에 기여한 경험이 있는 분
  • 정량 데이터와 VOC를 함께 보며 문제를 정의할 수 있는 분
  • 퍼널, 리텐션, 전환율, 세그먼트 분석에 대한 기본 이해가 있는 분
  • 단순 리포팅이 아니라 실제 제품과 비즈니스 문제를 해결하고 싶은 분
  • 여러 직군과 협업하며 분석 결과를 명확하게 전달할 수 있는 분
우대 사항
  • San Francisco, CA / 미국 근무가 가능한 분
  • 영상 서비스, 소셜미디어, UGC 서비스 분석 경험이 있는 분
  • 생성형 퍼널, 업로드 퍼널, 편집 퍼널, 공유 퍼널을 분석해본 경험이 있는 분
  • 피드, 알림, 댓글, 좋아요, 팔로우 등 커뮤니티 기능을 분석해본 경험이 있는 분
  • Retention, activation, new users 등 그로스 관련된 문제를 풀어본 경험
  • A/B 테스트 설계 및 결과 해석 경험이 있는 분
  • 이벤트 설계, 데이터 스키마 설계, 분석 환경 개선 경험이 있는 분
  • BI 대시보드 구축 및 운영 경험이 있는 분
  • 게이머이거나 게임 유저 행동에 대한 높은 이해가 있는 분
  • 사람들이 왜 콘텐츠를 만들고, 왜 공유하고, 왜 반응하고, 왜 다시 돌아오는지에 관심이 많은 분
이런 분과 잘 맞아요
  • 숫자를 정리하는 사람보다 문제를 푸는 사람
  • 데이터 뒤에 있는 유저의 행동과 맥락을 이해하려는 사람
  • 분석 결과를 실제 제품 변화로 연결하는 데 흥미가 있는 사람
  • 빠르게 질문을 세우고, 빠르게 검증하고, 빠르게 방향을 조정하는 환경을 선호하는 사람
  • 생성 행동과 커뮤니티 행동을 하나의 흐름으로 보는 데 흥미가 있는 사람
  • 전 세계 게이머가 더 쉽게 콘텐츠를 만들고 더 강하게 연결될 수 있도록 돕는 제품을 함께 만들고 싶은 사람
이력서에서 이런 점을 알려주세요
  • 지금까지 어떤 Product를 분석했고, 어떤 문제를 발견하고 해결했는지 알려주세요.
  • 데이터 분석을 통해 제품이나 비즈니스 의사결정에 어떤 영향을 만들었는지 구체적으로 적어주세요.
  • 퍼널, 리텐션, 실험 분석 등에서 어떤 인사이트를 만들었고 어떤 결과로 이어졌는지 알려주세요.
  • 가능하다면 수치 변화와 함께 설명해주세요.
  • 이벤트 설계, 데이터 품질 개선, 대시보드 구축 경험이 있다면 함께 적어주세요.
  • 업무와 경력의 나열보다 지원자님의 문제 정의, 고민, 액션, 학습, 성과가 드러나면 좋습니다.
